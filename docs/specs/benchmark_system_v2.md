# Benchmark System v2（ベンチマークシステム仕様・根拠付き）

## 1. 目的

本ドキュメントは、本プロジェクトで使用する フィジカル能力評価の
**ベンチマーク体系** を定義する。

本ツールは主にジュニアアスリート（U12）の育成支援を目的としているため、
成人トップアスリートの数値をそのまま評価基準として使うと
誤った解釈につながる可能性がある。

そのため本システムでは

「科学的知見に基づく原則」 ＋ 「本プロジェクトの設計判断」

の2つを分けて明記する。

------------------------------------------------------------------------

# 2. 科学的根拠（研究・ガイドライン）

以下は外部研究・ガイドラインに基づく考え方である。

## 2.1 ジュニア選手は成熟度の影響を強く受ける

若年アスリートの能力評価では

-   身長成長
-   PHV（Peak Height Velocity）
-   神経発達

の影響が大きい。

そのため

**成人トップ基準のみで能力評価するのは適切ではない。**

参考

-   NSCA Position Statement: Long-Term Athletic Development
-   IOC Consensus Statement on Youth Athlete Development

------------------------------------------------------------------------

## 2.2 単発の測定値よりトレンドが重要

ジュニア育成では

「現在の能力」より

**能力の成長速度**

が重要とされている。

理由

-   成長スパートによる能力変化
-   技術習得による急激な改善
-   神経系発達

参考

-   NSCA LTAD framework

------------------------------------------------------------------------

## 2.3 スプリント能力は成長とともに大きく変化する

研究では

-   sprint time
-   jump power

は思春期前後で大きく変化することが報告されている。

そのため

**年齢別 benchmark が必要になる。**

------------------------------------------------------------------------

# 3. 本プロジェクトの設計ルール

以下は本プロジェクト独自の設計判断である。

目的

-   スコアの直感的理解
-   レーダーチャート可視化
-   長期比較

------------------------------------------------------------------------

# 4. Benchmark Levels

4段階のベンチマークを採用する。

  Level         意味
  ------------- ------------
  Youth         年齢平均
  Competitive   競技レベル
  Elite         高水準
  World         世界トップ

------------------------------------------------------------------------

# 5. World Progress Model

スコアの正規化は次のスケールで行う。

  Level         Score
  ------------- -------
  floor         0
  competitive   50
  elite         75
  world         100

このスケールは以下の理由で採用した。

1.  レーダーチャートの可読性
2.  競技レベルの直感理解
3.  将来的な比較分析

------------------------------------------------------------------------

# 6. radar_score

各測定値は次の式で正規化する。

radar_score = 100 × (value - floor_anchor) / (world_elite_p50 -
floor_anchor)

制限

score \< 0 → 0\
score \> 100 → 100

------------------------------------------------------------------------

# 7. benchmark_values.csv

ベンチマークはデータとして管理する。

例

test,floor_anchor,competitive,elite,world_elite_p50

20m_sprint,4.2,3.6,3.3,3.1 vertical_jump,25,40,50,60

------------------------------------------------------------------------

# 8. Youth Benchmark の扱い

U12の評価では

world benchmark だけではなく

**youth benchmark を参考値として併用する。**

理由

-   成長差の影響
-   身長差
-   技術差

------------------------------------------------------------------------

# 9. 解釈方針

ジュニア選手では

単発の測定値より

**トレンド分析を重視する。**

例

-   月次改善
-   成長曲線
-   トレーニング効果

------------------------------------------------------------------------
