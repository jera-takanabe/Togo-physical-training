# 分析の妥当性レビュー

## 1. 目的

本資料は、U12 / 中学ラグビー選手向けフィジカル分析において、現在採用している6軸構成と各テスト項目のドメイン割り当てが妥当かを確認するためのレビュー資料である。

対象は backlog.md の以下項目とする。

- 2.1 6軸構成の妥当性確認
- 2.2 各テスト項目とドメイン対応の妥当性確認

なお、2.3 レーダーチャート不整合の修正は対応済みであり、本資料では対象外とする。

---

## 2. 現在の6軸構成

現在のレーダーチャートは以下の6軸で構成する。

| Axis | 日本語イメージ | 主に見たい能力 |
|---|---|---|
| Speed | スピード | 加速、疾走速度 |
| Power | 下半身パワー | ジャンプ、爆発的出力 |
| Elastic | 弾性・反発力 | 接地の短さ、反発利用 |
| COD | 方向転換 | 減速、切り返し、再加速 |
| Upper | 上半身出力 | 押す・投げる・コンタクト基礎 |
| Endurance | 持久・反復能力 | 高強度運動の維持、回復力 |

---

## 3. 6軸構成の妥当性レビュー

### 3.1 Speed

#### 評価したい能力

- 0〜10m の初速
- 0〜30m の加速能力
- 将来的には Fly 区間による最高速度寄りの能力

#### ラグビーとの関係

ラグビーでは、長距離の最高速度だけでなく、短い距離での加速、相手との間合いを詰める動き、抜け出しの初速が重要である。

#### 妥当性

Speed は独立軸として妥当。

#### 注意点

- 10m と 30m はどちらも Speed だが、同じ能力を完全に見ているわけではない
- 10m は初速寄り、30m は加速持続寄り
- Fly 区間を追加すると、最高速度寄りの評価が可能になる

---

### 3.2 Power

#### 評価したい能力

- 下半身の爆発的出力
- 垂直方向・水平方向への力発揮

#### ラグビーとの関係

タックル、コンタクト、加速、ジャンプ、ステップの土台になる。

#### 妥当性

Power は独立軸として妥当。

#### 注意点

- CMJ は下半身パワーを見る代表指標として妥当
- 立ち幅跳びを含める場合は、水平方向の出力として Speed / Power の中間的性質を持つ
- 体重の影響を受けるため、将来的には相対評価も検討余地あり

---

### 3.3 Elastic

#### 評価したい能力

- 接地時間の短さ
- 腱・筋の反発利用
- 連続ジャンプ能力
- RSI

#### ラグビーとの関係

スプリント、ステップ、切り返し、連続動作のキレに関係する。

#### 妥当性

Elastic は Power とは分けて扱う価値がある。

#### 注意点

- Power は「大きな力を出す能力」
- Elastic は「短時間で反発を使う能力」
- CMJ と RSI を同じ軸に入れると意味が混ざるため、Elastic 軸を独立させるのは妥当

---

### 3.4 COD

#### 評価したい能力

- 減速
- 切り返し
- 再加速
- 左右差
- 姿勢制御

#### ラグビーとの関係

ラグビーでは直線スピードだけでなく、相手をかわす、追う、止まる、再加速する能力が重要である。

#### 妥当性

COD は独立軸として妥当。

#### 注意点

- COD は Speed と強く関係するが、同一ではない
- 単純なスプリントが速くても、切り返しが苦手な選手は存在する
- プロアジリティは妥当だが、測定条件の統一が重要

---

### 3.5 Upper

#### 評価したい能力

- 上半身の押す力
- 投げる力
- 体幹を含む上半身出力
- コンタクトの基礎能力

#### ラグビーとの関係

タックル、ラック、ボールキャリー、ハンドオフ、パス動作の土台になる。

#### 妥当性

Upper はラグビー向け評価として妥当。

#### 注意点

- MBT は上半身だけでなく体幹・股関節の影響も受ける
- 座位 MBT にすると下半身の影響を抑えやすい
- 腕だけの能力ではなく、上半身出力の代表値として扱うのが自然

---

### 3.6 Endurance

#### 評価したい能力

- 高強度運動を繰り返す能力
- 回復力
- 試合後半まで動き続ける能力

#### ラグビーとの関係

ラグビーでは、単純な長距離持久力よりも、スプリント・コンタクト・移動を繰り返す能力が重要である。

#### 妥当性

Endurance は独立軸として妥当。

#### 注意点

- Yo-Yo IR1 は間欠的持久力の評価として有用
- RSA は repeated sprint ability に近く、Endurance と Speed の中間的性質を持つ
- 将来的には Endurance をさらに分解する余地があるが、現段階では1軸でよい

現時点では、`test_scores.csv` の `domain` 列は現行6軸そのものではなく、旧ドメイン名またはサブドメイン名に近い値を保持している。

- `domain_scores.csv` は将来的に `subdomain_scores` と `axis_scores` に分けることを検討する

## 5. 現時点の判断

現在の6軸構成は、U12 / 中学ラグビー選手のフィジカル評価として大きく破綻していない。

特に以下の点は妥当である。

- Speed と COD を分けている
- Power と Elastic を分けている
- Upper をラグビー向けの独立軸として持っている
- Endurance を単なる長距離走ではなく、間欠的持久力として扱っている

また、各テスト項目と現行6軸の対応についても、現時点では大きく破綻していない。

ただし、現在の `test_scores.csv` の `domain` 列は、現行6軸の `axis` ではなく、旧ドメイン名または `subdomain` に近い値を保持している。

そのため、今後の設計では以下を明確に分離する必要がある。

- test
- subdomain
- axis
- score layer

特に、`domain_scores.csv` は現在の名前のままだと、`subdomain` 集計なのか `axis` 集計なのかが分かりにくい。

将来的には、以下のように分けることを検討する。

- `test_scores.csv`
- `subdomain_scores.csv`
- `axis_scores.csv`
- `rugby_physical_score.csv`

## 6. 注意すべき論点

### 6.1 Power と Elastic の混同

CMJ、立ち幅跳び、RSI、pogo などは似て見えるが、評価したい能力が異なる。

- Power: 大きな力を出す能力
- Elastic: 短い接地で反発を使う能力

この2つを混ぜると、トレーニング判断が曖昧になる。

---

### 6.2 Endurance の意味づけ

Endurance は、長距離走のような持久力ではなく、ラグビーに必要な間欠的持久力として扱う。

そのため、評価項目は以下を優先する。

- Yo-Yo IR1
- RSA
- 高強度反復能力
- 回復力

---

### 6.3 1つのテストが複数能力を含む問題

一部のテストは、単一ドメインだけを測っているわけではない。

例:

- 立ち幅跳び: Power + Speed要素
- RSA average time: Endurance + Speed要素
- RSA decline: Endurance + 疲労耐性 / 回復力
- MBT: Upper + 体幹 / 姿勢保持要素
- Pro Agility / 5-10-5: COD + Speed + 減速 + 再加速 + 姿勢制御

ただし、現段階では1つのテストを複数axisへ配点せず、主axisを1つに決めて扱う。

理由は以下である。

- システムが複雑になりすぎることを避ける
- スコアの説明可能性を保つ
- 測定数が少ない段階では、細かい配点よりも継続測定と傾向把握を優先する
- 1つのテストを複数axisに配点すると、総合スコアで二重計上になる可能性がある

したがって、現時点では以下の方針とする。

| Test | 主axis | 副次的に含む要素 |
|---|---|---|
| `standing_long_jump` | Power | Speed / 水平方向加速 |
| `rsa_avg_time` | Endurance | Speed / 反復スプリント全体の走力 |
| `rsa_decline` | Endurance | 疲労耐性 / 回復力 |
| `medicine_ball_throw_2kg` | Upper | 体幹 / 姿勢保持 |
| `pro_agility_5_10_5` | COD | Speed / 減速 / 再加速 / 姿勢制御 |

## 7. 今後の改善候補

現時点では6軸構成を維持する。

ただし、将来的には以下を検討する。

- Fly区間をSpeed内の補助指標として追加
- RSI / pogo / DJ のどれをElastic代表指標にするか整理
- RSAをEnduranceに入れるか、Repeated Sprint Abilityとして別扱いするか検討
- MBTの測定姿勢を固定し、Upper指標としての再現性を高める
- 可動域、柔軟性、姿勢評価は現段階では追加しないが、将来的な拡張候補とする
